Tarea: Programación Orientada a Objetos
Nombre: Alisson Quiguango

Descripción:

Esta tarea contiene una serie de ejercicios enfocados en el uso de métodos `void`, métodos con `return` y métodos mixtos, aplicando conceptos fundamentales de la Programación Orientada a Objetos como encapsulamiento, validación de datos y lógica de negocio.

---

Ejercicio 1: Métodos `void` - Estudiante

Se creó una clase `Estudiante` que permite:

* Registrar nombre, carrera y edad
* Mostrar los datos ingresados

- Métodos usados:

* `registrarDatos()` → ingreso de datos
* `mostrarDatos()` → salida de información

---

Ejercicio 2: Factura de Producto (`void`)

Se desarrolló una clase que:

* Registra nombre, precio y cantidad
* Calcula el total
* Aplica descuento del 10% si supera 1000

- Métodos:

* `ingresarDatos()`
* `mostrarFactura()`

---

Ejercicio 3: Métodos con `return` - Calificación

- Clase que:

* Calcula el promedio de 3 notas
* Determina si el estudiante aprueba o reprueba

- Métodos:

* `calcularPromedio()`
* `obtenerEstado()`
* `mostrarReporte()`

---

Ejercicio 4: Planilla de Luz (`return`)

- Se implementó:

* Cálculo de subtotal
* Cálculo de IVA (15%)
* Total a pagar

- Métodos encadenados:

* `calcularSubtotal()`
* `calcularIVA()`
* `calcularTotal()`

---

Ejercicio 5: Métodos mixtos - Empleado

- Clase que:

* Calcula sueldo base
* Aplica bono
* Determina sueldo final

- Uso combinado:

* `void` → entrada y salida
* `return` → cálculos

---

Ejercicio 6: Pedido Restaurante

- Sistema que:

* Calcula subtotal
* Aplica descuento del 10% si supera 25
* Muestra total final

---

Ejercicio 7: Control de Inventario

- Sistema más completo que:

* Maneja productos con encapsulamiento
* Valida precio y stock
* Calcula valor del inventario
* Determina estado del stock:

  * Stock suficiente
  * Requiere reposición
  * Producto agotado

- Conceptos aplicados:

* Atributos privados
* Métodos `get` y `set` con validación
* Constructor
* Lógica de negocio
